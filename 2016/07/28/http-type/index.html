HTTP请求方法

发表于 2016-07-28   |   分类于 HTTP   |  

GET

get方法是请求指定的资源，一般用于读取数据。比如页面js、image、css、后台数据都是通过get方法请求获得。

HEAD

head和get方法都是请求指定的资源，用于读取数据，但是head方法请求不会回传内容，即响应内容只有状态码、消息报头，没有消息体。这个作用一般是用于查看服务器消息

POST

post方法是向服务器提交数据，请求服务器处理。一般作用是数据提交、文件上传等等。根据restful协议，post方法一般是新建数据

PUT

put方法也是向服务器提交最新数据，请求服务器处理，但是他的作用是修改数据

DELETE

delete方法是请求服务器删除uri所标识的资源，delete请求后，指定资源就会被删除

OPTIONS

options方法和head方法类似，一般也是查看服务器性能。这个方法向服务器发送请求后，服务器会返回所支持的所有请求方法，options方法请求的资源将会已*号资源名称。向服务器发送options请求，可以判断服务器是否可以正常工作,这种请求方式叫做嗅探请求。javascript中xmlhttprequest对象进行cros跨域请求资源就是通过options方法嗅探请求，借此判断是否有访问该资源的权限。

CONNECT

connect链接可以将链接方式改为管道方式代理服务器，通常用于SSL加密服务器的链接与非加密的HTTP代理服务器的通信。

TRACE

trace请求服务器，然后服务器显示请求的信息。这个一般用于测试、诊断接口

PATCH

patch方法和put方法类似，都是向服务器提交最新的资源，请求服务器更新资源。方法区别就是patch方法是更新一部分内容，put方法是更新所有内容；patch方法如果没有查找到要更新的内容会重新插入一条新纪录，put方法则不会。
