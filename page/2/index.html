api接口开发要做的那几件事

发表于 2016-07-23   |   分类于 PHP ， API   |  

前后端分离

开发一般都是前后端分离来提高开发效率。我在开发过程中常用到的前后端分离有两种。

  1. 传统MVC模式。直接通过model层，把数据分配到view，然后通过模板引擎去解析生成html文件，最后通过服务器返回给浏览器。这样的开发模式缺点很明显，首先前端人员还要学习模板渲染的语法，其次性能也不好。
  2. 为了使数据展现过程更加直接，并且提供更好的用户体验，就有大神对MVC模式进行改进。首先从浏览器发送AJAX请求，然后服务端接受该请求并返回JSON数据返回给浏览器，最后在浏览器中进行界面渲染。这就是RESTful架构。

REST全称是Representational State Transfer（表述性状态转移），是Roy Thomas Fielding在他2000年的博士论文中提出的。

YAML 语言简介

发表于 2016-07-23   |   分类于 script ， yaml   |  

YAML是”YAML Ain’t a Markup Language”（YAML不是一种置标语言）的递归缩写。
YAML 的意思其实是：”Yet Another Markup Language”（仍是一种置标语言）

编程免不了要写配置文件，怎么写配置也是一门学问。
YAML 是专门用来写配置文件的语言，非常简洁和强大，远比 JSON 格式方便。

拓展ci的model

发表于 2016-07-22   |   分类于 PHP ， CodeIgniter   |  

作用

CI框架已经提供了ORM操作了，为什么我还去要拓展ci的model呢？主要是基于以下几点考虑的。

  • 因为现在对数据库的操作都封装model里面，没必要每个model都去加载database类，以后维护也麻烦。
  • CI因为要增加灵活，导致对表的操作都要传递表名，觉得很蛋疼。对数据库的操作都是在model里面完成，所以没必要传递了，只要model和表名之间做个映射就好啦。
  • 现在都是团队开发，每个人划分模块开发，各人负责自己的model开发，如果不规范增删改查的，那会增加很多工作量，假如某个人要调用别人的模块，还要阅读别人的模块api。
