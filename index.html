MySQL事务

发表于 2016-07-27   |   分类于 MySQL   |  

所谓事务，它是一个操作序列，这些操作要么都执行，要么都不执行，它是一个不可分割的工作单位。例如，银行转帐工作：从一个帐号扣款并使另一个帐号增款，这两个操作要么都执行，要么都不执行。

数据库事务必须具备ACID特性，ACID是Atomic（原子性）、Consistency（一致性）、Isolation（隔离性）和Durability（持久性）的英文缩写。

MySQL存储引擎

发表于 2016-07-26   |   分类于 MySQL   |  

存储引擎：就是如何存储数据，如何为存储的数据建立索引和如何更新、查询数据等技术的实现方法（存储管理机制）。oracle和sql server数据库都是只有一种存储引擎，而mysql数据库提供了多种存储引擎。

MyISAM

  • 静态MyISAM：数据表中的列长度固定好（没有varchar、text、blob数据类型）的时候，用这种数据引擎。好处就是处理数据的效率高，如果数据损坏，修复比较容易。
  • 动态MyISAM：数据库表中自动选择列长的时候，自动切换到此引擎。好处存储空间比静态MyISAM引擎小。
  • 压缩MyISAM：原来压缩是MyISAMCHK工具压缩，更换压缩方式，减小存储空间，但是数据不能修改，读取时因为需要先解压，所以读取速度比较慢。

工厂模式（factory modal）：就是统一实例化。方便日后修改，比如初始化一个类的时候，可能刚开始需要传递两个参数，假如日后初始化的时候需要修改成为三个参数，那么修改的时候，只需要在工厂类中修改就可以，不用到每个实例化的控制器中修改。

api接口开发要做的那几件事

发表于 2016-07-23   |   分类于 PHP ， API   |  

前后端分离

开发一般都是前后端分离来提高开发效率。我在开发过程中常用到的前后端分离有两种。

  1. 传统MVC模式。直接通过model层，把数据分配到view，然后通过模板引擎去解析生成html文件，最后通过服务器返回给浏览器。这样的开发模式缺点很明显，首先前端人员还要学习模板渲染的语法，其次性能也不好。
  2. 为了使数据展现过程更加直接，并且提供更好的用户体验，就有大神对MVC模式进行改进。首先从浏览器发送AJAX请求，然后服务端接受该请求并返回JSON数据返回给浏览器，最后在浏览器中进行界面渲染。这就是RESTful架构。

REST全称是Representational State Transfer（表述性状态转移），是Roy Thomas Fielding在他2000年的博士论文中提出的。

拓展ci的model

发表于 2016-07-22   |   分类于 PHP ， CodeIgniter   |  

作用

CI框架已经提供了ORM操作了，为什么我还去要拓展ci的model呢？主要是基于以下几点考虑的。

  • 因为现在对数据库的操作都封装model里面，没必要每个model都去加载database类，以后维护也麻烦。
  • CI因为要增加灵活，导致对表的操作都要传递表名，觉得很蛋疼。对数据库的操作都是在model里面完成，所以没必要传递了，只要model和表名之间做个映射就好啦。
  • 现在都是团队开发，每个人划分模块开发，各人负责自己的model开发，如果不规范增删改查的，那会增加很多工作量，假如某个人要调用别人的模块，还要阅读别人的模块api。
